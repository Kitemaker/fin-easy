Token Vault 4 / 10

Auth0 Token Vault

Your tokens stay in Auth0.
Not in FinEasy.

Token Vault stores per-service OAuth tokens inside Auth0, encrypted in your user metadata. FinEasy fetches them at runtime for each tool call — and only if you've connected that service.

🔐
Zero token storage in FinEasy

App DB holds no OAuth tokens — only user IDs and connection flags.

🗂️
Per-service isolation

Bills token is completely separate from the Investments token — a breach of one exposes nothing else.

⚡
Instant revocation

Delete a token in Auth0 and the agent immediately loses that service — no app deploy needed.

Fetch write-scoped token from vault const token = await getServiceToken( userId, 'bills' ); if (!token) return notAuthorized('Bills'); // 2. Key design decisions

No chart library dependency

Investment donut chart, spending bar chart, and bills timeline are all pure SVG — keeping the bundle lean and customizable.

Write scope explicit opt-in

The Bills service token is the only write-scoped token. Checking, Savings, and Investments tokens are read-only at the vault level.

Direct data layer calls in tools

Tools call mock-data functions directly rather than making HTTP self-calls — eliminating race conditions in streaming responses.

Per-tool token fetch

Each tool call independently retrieves its token from the vault — no shared token state, no risk of cross-service token leakage.
